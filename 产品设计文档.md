## 一、产品基本信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 小米AIoT智能植物养护助手 |

## 四、通信协议

### 4.1 U1P ↔ U2P 通信（C2 ZigBee）

```
帧格式 (二进制):
┌──────┬──────┬──────┬──────┬──────────────────┐
│ 0xFC │ Len  │ 0x01 │ Mode │     Data         │
└──────┴──────┴──────┴──────┴──────────────────┘
```

- **帧头**: `0xFC` — 标识数据帧开始
- **Len**: 数据长度（含 Mode + Data）
- **0x01**: 固定标识
- **Mode**: `0x01` 广播模式
- **Data**: 实际数据内容

### 4.2 数据格式

**U2P → U1P（植物识别结果）**:
```
植物名|置信度
示例: 绿萝|0.95
     多肉|0.87
     发财树|0.92
```

**U1P → U2P（浇水提醒）**:
```
WATER|植物名|温度|湿度|光照
示例: WATER|绿萝|28.5|65.0|1200
```

### 4.3 C2 ZigBee 初始化流程（5步）

```
Step1: 读取设备信息   → 0xFE 0x01 0xFE 0xFF
Step2: 设置工作模式   → 0xFD 0x02 0x01 {终端/协调器} 0xFF
Step3: 设置PAN ID    → 0xFD 0x03 0x03 0x2C 0x3F 0xFF
Step4: 设置网络组    → 0xFD 0x02 0x09 0x01 0xFF
Step5: 设置加密密钥  → 0xFD 0x11 0x04 ... 0xFF
```

- U1P 侧 C2 初始化为 **终端模式 (TERMINAL)**
- U2P 侧 C2 初始化为 **协调器模式 (COORDINATOR)**

## 五、核心功能

### 5.1 传感器采集

| 传感器 | 测量范围 | 精度 | 采集间隔 |
|--------|---------|------|---------|
| SHT35 温度 | -40~125°C | ±0.2°C | 1秒 |
| SHT35 湿度 | 0~100%RH | ±1.5%RH | 1秒 |
| BH1750 光照 | 1~65535 lux | ±20% | 1秒 |

### 5.2 环境评分算法

加权计算公式：
```
总分 = 温度得分 × 30% + 湿度得分 × 40% + 光照得分 × 30%
```

| 维度 | 最优区间 | 扣分规则 | 权重 |
|------|---------|---------|------|
| 温度 | 依植物档案（如绿萝18~28°C） | 每偏离1°C扣10分 | 30% |
| 湿度 | 依植物档案（如绿萝50~70%） | 每偏离5%扣2分 | 40% |
| 光照 | 依植物档案（如绿萝200~500lux） | 每偏离50lux扣0.5分 | 30% |

评分范围：0~99分，数码管显示为 `评分 + 55`（即显示 55~154）

### 5.3 自动遮阳控制

执行器：步进电机窗帘（E3模块）

| 条件 | 动作 |
|------|------|
| 光照 > curtain_close_threshold | 关闭窗帘（遮阳） |
| 光照 < curtain_open_threshold | 打开窗帘 |
| 回差区间内 | 保持当前状态，避免频繁启停 |

各植物阈值：
- 绿萝：关 800 lux / 开 400 lux
- 多肉：关 1200 lux / 开 600 lux
- 发财树：关 800 lux / 开 400 lux

### 5.4 自动通风控制

执行器：直流风扇（E2模块，PWM调速）

| 条件 | 动作 |
|------|------|
| 湿度 > fan_open_threshold | 开启风扇 90% 转速 |
| 湿度 < fan_close_threshold | 关闭风扇 |
| 回差区间内 | 保持当前状态 |

各植物阈值：
- 绿萝：开 80% / 关 70%
- 多肉：开 70% / 关 60%
- 发财树：开 80% / 关 70%

### 5.5 蒸发指数与浇水提醒

蒸发指数计算：
```
蒸发指数 = 温度 × 0.3 + 光照 × 0.5 - 湿度 × 0.2
```

- 每秒累加蒸发指数到 `evaporation_sum`
- 当累计值超过 **500** 时触发浇水提醒
- 触发后：U1P通过C2广播 `WATER|植物|温|湿|光` 给U2P
- U2P收到后调用 **Server酱** 推送微信消息给用户
- 触发后进入 3600秒冷却期

### 5.6 植物档案切换

支持三种植物自动切换：

| 植物 | 适温 | 适湿 | 适光 |
|------|------|------|------|
| 绿萝 | 18~28°C | 50~70% | 200~500 lux |
| 多肉 | 15~25°C | 30~50% | 500~1000 lux |
| 发财树 | 20~30°C | 40~60% | 300~600 lux |

切换方式：
- **手动**：按 SW2 按键循环切换
- **自动**：U2P识别到植物后通过C2通知U1P自动切换

### 5.7 RGB LED 指示

| 评分区间 | 颜色 | 含义 |
|---------|------|------|
| 90~99分 | 绿色 | 环境优秀 |
| 70~89分 | 黄色 | 环境良好 |
| 50~69分 | 橙色 | 环境一般 |
| 0~49分 | 红色 | 环境较差 |

按键 SW1 可切换为固定颜色模式（红/绿/蓝/紫循环）。

## 六、按键功能

| 按键 | 功能 |
|------|------|
| SW1 | 切换RGB模式：自动(跟随评分) → 红色 → 绿色 → 蓝色 → 紫色 |
| SW2 | 切换植物档案：绿萝 → 多肉 → 发财树，数码管显示编号(1~3) |
| SW3 | 手动开关窗帘：0% ↔ 100% 切换 |

## 七、视觉识别流程（U2P）

### 7.1 植物识别流程

```
1. USB摄像头拍照（间隔10秒）
2. 调用百度AI植物识别API
3. 筛选目标植物（绿萝/多肉/发财树）
4. 置信度 ≥ 30% 时通过C2广播给U1P
5. 连续广播30次（15秒，每500ms一次）
6. 识别成功后停止拍照，仅监听浇水提醒
```

## 九、产品亮点

1. **ZigBee无线通信**：使用C2 ZigBee模块实现U1P与U2P之间的无线数据传输，自组网、低功耗、通信稳定
2. **多传感器融合评分**：基于SHT35和BH1750数据，采用加权评分算法实时评估植物生长环境
3. **智能自动控制**：窗帘回差控制和风扇回差控制，避免频繁启停，延长执行器寿命
4. **AI视觉识别**：U2P侧通过百度AI识别植物种类，自动切换养护参数
5. **蒸发指数预测**：创新性引入蒸发指数模型，提前预判植物需水状态
6. **微信远程推送**：通过Server酱将浇水提醒推送到手机微信
